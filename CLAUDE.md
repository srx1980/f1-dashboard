AGENTS.md — F1 Dashboard

> Read this first. This file tells you (Cursor) everything you need to know to build the frontend for the F1 Dashboard. No guessing, no assumptions. Build exactly what's described here.

## What We're Building

A clean, minimal, English-language F1 dashboard — a single-page site that gives F1 fans a quick overview of the current season. No news clutter, no ads, no paywalls. Just the data that matters.

The main dashboard stays a single page. Alongside it, the site now also serves **standalone static pages** (one per driver, one per team, and a 2026 regulations page) whose only purpose is SEO / organic traffic. These are separate files, generated/authored once and linked from the dashboard; they do not change the single-page nature of the dashboard itself.

**Target audience:** Global F1 fans who want standings, results, and schedule at a glance.

## Operations — Hermes AI Agent

The live site is **managed by an AI agent called Hermes** (the data pipeline). This split is intentional:

| Layer | Owner | What it does |
| --- | --- | --- |
| **Frontend** | You (Cursor) / humans | `index.html`, `css/style.css`, `js/main.js` — static HTML/CSS/JS |
| **Data** | **Hermes** (AI agent) | Fetches F1 data, writes `/data/*.json` and optional `summary.md`, commits & pushes to the Git repo |
| **Hosting** | GitHub Pages | Serves the repo as static files — no server, no build step |

**Hermes responsibilities:**
- Update `data/standings.json`, `data/schedule.json`, `data/results.json` after each race / on schedule
- Append the just-finished race to `data/season_results.json` (the season-long archive that powers the per-driver "Results" table), then run `python scripts/generate_pages.py` to rebuild the driver/team pages + sitemap
- Update the Race Summary section in `index.html` directly (static HTML, for SEO)
- Push changes to the Git repository

**Hermes does NOT touch:** frontend HTML/CSS/JS (unless explicitly asked).

**Frontend data loading:** `main.js` fetches JSON with `{ cache: 'no-cache' }`, so Hermes updates appear on the next page load without cache-busting the data files. Asset cache-busting (`?v=N` on CSS/JS in `index.html`) is only needed when the frontend code changes.

## Architecture (CRITICAL)

This is a fully static site with zero backend:

```
/data/*.json   ← Hermes writes these (NOT your job)
/index.html    ← You build this (static HTML/CSS/JS)
/js/main.js    ← Fetches /data/*.json and renders everything
/css/style.css
```

The site fetches JSON files from `/data/` at runtime using `fetch()`. You do NOT hardcode any data. The JSON files are generated by Hermes and pushed to the repo alongside your frontend. GitHub Pages serves everything as static files.

**Your job:** Build the HTML, CSS, and JS that reads those JSON files and renders a beautiful dashboard.

## Hard Constraints

1. **No frameworks.** Vanilla HTML, CSS, and JavaScript only. No React, Vue, Svelte, Tailwind, Bootstrap, etc.
2. **No build step.** No npm, no webpack, no bundler. Just plain files that work when opened or served.
3. **GitHub Pages compatible.** Everything must work when served from a static file server. No server-side code.
4. **Mobile-first responsive.** Must look great on phones, tablets, and desktop.
5. **Dark theme.** F1 aesthetic — dark backgrounds, bold typography, accent colors.
6. **Single-page dashboard.** The main dashboard keeps all season data on one scrollable page with anchor-link navigation. (Standalone SEO pages — driver/team/regulations — are separate static files, but the dashboard itself is never split into multiple pages.)

## Pages / Sections

The page has 6 sections stacked vertically (Upcoming Race is **above** Standings):

### 1. Hero / Header (sticky)
- Brand mark (inline SVG speedometer, NOT the official F1 logo) + "F1 Dashboard" title
- Current season year (e.g. "2026 Season")
- Navigation: anchor links to Upcoming, Standings, Results, Calendar
- Last updated timestamp (from `standings.json`, relative time)

### 2. Upcoming Race
- Race name, round number, Sprint badge if applicable
- Circuit name, city, country (no flag emojis)
- Live countdown to race start (client-side JS, updates every second)
- Caption: "until lights out · [UTC time]" → "LIGHTS OUT 🏁 / Race in progress" at T-0
- **Weekend schedule:** Qualifying and Race only — dates/times in **UTC** (English weekday, 24-hour, labeled `UTC`)

### 3. Season Standings
Two tables side-by-side (stack on mobile):

**Driver Standings:** Pos | Driver | Team | Pts | Wins  
**Constructor Standings:** Pos | Team | Pts | Wins

- Position column: plain numbers (1, 2, 3…) — no medal emojis
- Driver cell: full name + 3-letter code — **no nationality flags**
- Team-color left accent on each row + mini relative points bars
- "Last updated: Round X" subtitle

### 4. Latest Race Results
- Race name, date, circuit in subtitle
- Full results table: Pos | Driver | Team | Time/Status | Grid | Fastest Lap
- Position: numbers only (DNF styled for retirees)
- Driver: name + code, no flags
- Winner row: subtle gold background highlight; 🏆 trophy before finish time in Time column
- Grid → finish position-change indicator (▲ / ▼ / ▬)
- DNF status in distinct red style

### 5. Season Calendar
| Rnd | Date | Race | Circuit | Winner |

- No country flag emojis
- Completed races: green tint + checkmark on winner
- Next upcoming race: red left accent + "Next" tag
- Winner column: name or TBD

### 6. Race Summary (static HTML)
- AI-generated race summary written directly as static HTML in `index.html` (inside `#summary-content`)
- Hermes updates this section after each race — no JS fetch, no `summary.md` needed
- Content is always visible (no `is-hidden` class) and fully indexed by search engines for SEO

### `/data/results.json`
```json
{

  "raceName": "Monaco Grand Prix", "date": "2026-05-31",
  "circuit": {
    "circuitName": "Circuit de Monaco",
    "location": { "locality": "Monte Carlo", "country": "Monaco" }
  },
  "results": [
    {
      "position": "1", "grid": "2",
      "driver": { "code": "ANT", "givenName": "Andrea Kimi", "familyName": "Antonelli", "nationality": "Italian" },
      "constructor": "Mercedes", "laps": "78", "time": "1:42:13.847",
      "status": "Finished",
      "fastestLap": { "rank": "1", "lap": "72", "time": "1:16.234" }
    }
  ]

```

### `/data/season_results.json`
Season-long archive that powers the **per-driver "Results" table** on `drivers/<code>.html`. It is simply an array of the same objects as `results.json` — one per completed round. After each GP, Hermes appends the new `results.json` object to `races[]` (keep them ordered by `round`), then runs `scripts/generate_pages.py`.
```json
{
  "season": "2026",
  "races": [
    {
      "round": "9",
      "raceName": "British Grand Prix",
      "date": "2026-07-05",
      "circuit": { "circuitName": "Silverstone Circuit", "Location": { "locality": "Silverstone", "country": "UK" } },
      "results": [
        {
          "position": "1", "grid": "2",
          "driver": { "code": "LEC", "givenName": "Charles", "familyName": "Leclerc", "nationality": "Monegasque" },
          "constructor": "Ferrari", "status": "Finished", "time": "1:27:11.335",
          "fastestLap": { "rank": "8", "lap": "43", "time": "1:32.871" }
        }
      ]
    }
  ]
}
```
The generator reads, per driver, `grid`, `position`, `status`, and `time` for each round. Missing file → the "Results" section shows a friendly placeholder (no crash).

## JavaScript Requirements

`main.js` must:

1. `fetch()` all 3 JSON files from `/data/` in parallel on page load (`cache: 'no-cache'`)
2. Error handling: missing file → graceful message, don't break the page
3. Countdown timer: live countdown to next race, update every second
4. **Session times in UTC** — `fmtUTC()` with `timeZone: 'UTC'`, English weekday, 24-hour, `UTC` suffix
5. DNF highlighting: non-"Finished" status → distinct style
6. Smooth scrolling for anchor links
7. "Last updated" display: relative time from `standings.json`
8. Zero external dependencies

**Do NOT implement (removed by design):**
- Nationality / country flag emojis
- Podium medal emojis (🥇🥈🥉) — use plain position numbers
- Championship-leader badge in the hero
- Full FP1/FP2/FP3/Sprint schedule in the UI (Qualifying + Race only)
